Cloud Portfolio • Project 01

AWS Static Website – S3 + CloudFront (IaC with Terraform)

Deployed a globally available static website using Amazon S3 for storage and Amazon CloudFront as a CDN, fully provisioned with Terraform and secured with HTTPS using the default CloudFront certificate.

What this project shows
Amazon S3 Website Hosting Amazon CloudFront CDN Terraform (IaC) HTTPS by default
  • Infrastructure defined as code: S3 bucket, website configuration, public-read policy, and CloudFront distribution are all created via Terraform.
  • CloudFront serves content over HTTPS with global edge locations, improving latency and security.
  • Simple, low-cost architecture suitable for portfolios, landing pages, or documentation sites.
Tech stack
  • AWS: S3, CloudFront
  • Infrastructure as Code: Terraform
  • Language: HTML / CSS (no framework)
  • Region: ca-central-1 (Canada)
Request flow
1. Browser → User visits the CloudFront URL (https://<distribution>.cloudfront.net).
2. CloudFront (CDN) → Terminates HTTPS, caches static assets, and forwards cache misses to the S3 website endpoint.
3. S3 Static Website → Hosts index.html and 404.html objects and returns responses.
4. Terraform → Manages all infrastructure lifecycle: init, apply, and destroy.

Project 03 — Billing & Cost Monitoring

AWS Billing Dashboard & Budget Alerts

This project focuses on monitoring AWS costs using the built-in Billing & Cost Management tools. I created reusable Cost Explorer reports and budget alerts to track monthly spend and avoid surprises while staying inside the Free Tier.

What this project shows
AWS Billing & Cost Management Cost Explorer Reports AWS Budgets Email Alerts
  • Created a “AWS Monthly Billing Overview” Cost Explorer report to see total monthly costs, grouped by service, across the last 6 months.
  • Added a “Daily Spend (Last 14 Days)” report to quickly spot any unusual jumps in daily AWS usage.
  • Configured an AWS Budget with thresholds (for example, 50% and 100% of the monthly budget) that send email alerts.
  • Verified that portfolio resources (S3, CloudFront, EC2, Lambda) remain aligned with the Free Tier wherever possible.
Tech stack
  • AWS Billing & Cost Management console
  • AWS Cost Explorer (saved reports)
  • AWS Budgets (email notifications)
  • Region: ca-central-1 (Canada)
Monitoring flow
1. Cost Explorer → Saved reports visualize monthly and daily AWS spend by service.
2. Budget → Watches actual and forecasted costs against a monthly limit (for example, $10).
3. Alerts → When thresholds are crossed, AWS sends notification emails to my account.
4. Action → If needed, I can shut down or right-size resources (EC2, volumes, etc.) to stay on budget.
Project 04 — EC2 Web Server in a VPC

Amazon EC2 + Nginx (Public Web Server)

Here I re-hosted the portfolio site from Project 01 on an Amazon EC2 instance running Nginx inside a custom VPC. This demonstrates the classic “VM + web server” pattern with proper networking and secure access via Systems Manager.

What this project shows
Amazon EC2 Nginx Web Server Custom VPC + Subnet Security Groups SSM Session Manager
  • Launched an Amazon Linux 2 EC2 t2.micro instance in a public subnet of my own VPC.
  • Attached a security group allowing HTTP (TCP/80) from the internet, with admin access handled via Session Manager.
  • Installed and configured Nginx to serve the portfolio index.html from /usr/share/nginx/html.
  • Verified that visiting the instance’s public IP returns the same portfolio content as the S3 + CloudFront site.
Tech stack
  • AWS: EC2, VPC, Subnets, Route Tables, Internet Gateway
  • Compute: Amazon Linux 2, Nginx
  • Access: AWS Systems Manager Session Manager
  • Region: ca-central-1 (Canada)
Network & request flow
1. Browser → EC2 → User visits http://<ec2-public-ip>.
2. Security Group → Inbound rule allows TCP/80 from the internet; other ports are restricted.
3. VPC Public Subnet → Route table sends 0.0.0.0/0 to the Internet Gateway so the instance is reachable.
4. Nginx on EC2 → Nginx serves the static portfolio files and returns responses to the client.
